Introduction

Welcome to the Fulvo Database Web UI. This is a web application that allows users to interact with an Oracle database with populated (fake) data. The database consists of seven different tables: Users, Players, Referees, Teams, House_League, Match_Stats, and Match_Referee. Each table has its own unique attributes and relationships with other tables. The goal of this assignment is for users to be able to perform CRUD operations on each table, as well as perform specific SQL queries on the database with user functionality Before getting started, this oracle database can only be accessed with a specific VPN connection by Toronto Metropolitan University. If you want to see this project in action, but don't have access to the VPN, please check the GitHub README for a video demonstration, as that is the only way (that I know of) to be able to utilize this application. Now let us get into the details of this application.

Application Details

The database was created using PL/SQL (Oracle SQL), with separate SQL scripts for creating, populating, and querying. We executed these scripts with Oracle SQL*Plus and Oracle SQL Developer. As for the web interface, we decided to use Python and Flask as our backend framework, with SQLAlchemy as our ORM (Object Relational Mapper). We also used Jinja2 as our a templating engine (as it comes with Flask), and jQuery is utilized in the frontend for interacting with the backend and adding functionality. As for our styling, we combined custom CSS and Bootstrap to create a clean and simple design.

User Requirements

  1. Perform certain SQL queries (for example, select all that are skill level advanced)
  2. Create, read, update, and delete records for each table
  3. Search for specific records for each table

Click on each item on the navbar to be able to visualize these requirements.
